## 一、优先级与轮次说明

| 字段 | 含义 |
|------|------|
| **P0** | 核心路径；失败则功能不可用或数据不可信 |
| **P1** | 重要功能与数据准确性；应在第一轮完整覆盖 |
| **P2** | 体验、边界、兼容性 |
| **阻塞** | **是**：不通过则后续模块无法有效验证（菜单/权限/查询数据集/组件配置基线） |
| **首轮冒烟 ★** | 第一轮冒烟测试必跑 |
| **准入 ★★** | 提测准入门禁；全部通过方可进入全量回归 |

> **口径差异待确认（测前与产品对齐）**  
> - 图一筛选项为「跟进人」；图二逻辑为「负责人」— 用例按**同一字段**验证，UI 文案以交互稿为准。  
> - 图一时间项：近30天 / 近90天 / 自定义；图二备注含近7/30/60/90天，默认**本月**；HTML 原型为「本月/上个月/本季度/近90天」— 以**最终交互稿选项列表**为预期，下列用例中「时间选项」步骤需替换为定稿列表。  
> - 图一 0511 新增「线索所在国家」；HTML 原型未体现 — **以图一/交互稿为准**单独验收。

## 二、测试数据准备清单（执行前必须就绪）

### 2.1 账号与权限

| 编号 | 数据类型 | 说明 | 用途 |
|------|----------|------|------|
| ACC-01 | **全量权限账号** | 后台为「公司特殊员工」单独配置：可查看线索看板**全部**数据（不限部门） | 数据准确性基准、交叉校验 |
| ACC-02 | **商城账户** | 手机号 `18143073992`，已开通线索看板菜单 | 特殊员工配置验收、冒烟主账号之一 |
| ACC-03 | **部门负责人** | 仅本部门及下级线索权限 | 部门筛选 + 权限边界 |
| ACC-04 | **普通销售** | 仅本人负责线索 | 负责人筛选 + 权限边界 |
| ACC-05 | **无菜单权限账号** | CRM 其他角色，未分配「线索看板」 | 菜单与路由拦截 |
| ACC-06 | **无数据权限账号** | 有菜单但权限范围内无线索 | 空态与 0 值展示 |

### 2.2 组织与人员

| 编号 | 数据类型 | 说明 |
|------|----------|------|
| ORG-01 | 部门树 | 至少 3 级：如「销售中心 → 销售一部 / 销售二部 → 一组」 |
| ORG-02 | 重名员工 | 2 名不同部门员工同名（如「张伟」） | 负责人关键词匹配 |
| ORG-03 | 离职/停用员工 | 历史线索负责人为停用账号 | 下拉与统计是否包含 |
| ORG-04 | 员工清单 | 每部门 ≥3 名在职销售，且各有分配线索 | 人员处理模块 |

### 2.3 线索业务数据（建议通过 API/脚本批量造数，并导出 ID 清单备查）

| 编号 | 数据类型 | 数量建议 | 关键字段/场景 |
|------|----------|----------|----------------|
| LEAD-01 | **时间跨度线索** | ≥200 条 | 创建时间分布在：本月、上月、近30天外、近90天边界前后各 5 条 |
| LEAD-02 | **部门分布** | 每部门 ≥50 条 | `部门` 分属销售一部、销售二部 |
| LEAD-03 | **负责人分布** | 每销售 ≥30 条 | 当前负责人、历史曾负责人（用于流转口径） |
| LEAD-04 | **国家/地区** | 各 ≥40 条 | `中国大陆` / `海外`（0511 新增筛选项） |
| LEAD-05 | **来源多样** | 每来源 ≥10 条 | 官网留资、百度推广、线下展会等 ≥5 种来源 |
| LEAD-06 | **状态分布** | — | 待跟进、跟进中、已转换、已作废均有 |
| LEAD-07 | **等级分布** | — | A/B/C/D/空 均有（来源与状态质量图表） |
| LEAD-08 | **公海流转** | ≥80 条事件 | 本期移入、自动回收、认领、当前仍在公海 |
| LEAD-09 | **人员处理事件** | ≥100 条 | 分配、首跟（24h 内/外）、转客户、主动/自动入公海 |
| LEAD-10 | **活动记录** | ≥50 条 | 普通电话、系统外呼、微信等类型 |
| LEAD-11 | **边界数据** | 各 1～3 条 | 作废率>30% 来源、零线索部门、零线索负责人 |
| LEAD-12 | **权限隔离数据** | — | 销售 A 仅能见自己线索；管理员能见全部 — 用于验证「权限过滤后数据集」 |

### 2.4 预期基准表（数据准确性必备用）

由开发或 BI 提供 **与筛选条件绑定的 SQL/报表导出**，至少包含：

1. 默认条件（本月 + 全公司 + 无负责人）下：三大类 KPI 各指标期望值  
2. 组合条件样例 ≥5 组（见 § 六 数据准确性用例）  
3. 各明细表行级汇总可与 KPI/图表互证

### 2.5 环境与配置

| 编号 | 项 | 说明 |
|------|-----|------|
| ENV-01 | 浏览器 | Chrome 最新、Edge；分辨率 1920×1080、1366×768 |
| ENV-02 | 组件配置 | 测试前清除浏览器 localStorage 中看板配置键（或提供后台重置），避免污染 |
| ENV-03 | 特殊员工配置入口 | 确认后台「公司特殊员工 / 线索看板全量权限」配置页面可用 |
